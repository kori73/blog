Probabilistic Computation 2: Rejection Sampling

Dec 7, 2020 â€¢ 3 min read

Rejection sampling is an exact sampling method that allows us to generate samples from a density $p(x)$ or an unnormalized density. Rejection sampling requires that we have another density $q(y)$ which we can sample from an. For rejection sampling to work, there must be a constant $M$ value ($M > 0$) that will satify the condition $Mq(x) \geq f(x)$ for all $x$.

The algorithm

  1. Generate a sample $y$ from $q(y)$.
  2. Generate a sample u from $U(0, 1)$.
  3. If $u \leq \frac {f(x)} {Mg(x)}$ accept the sample, otherwise reject it.

But what is the point of this $M$? Well, with this $M$ value, we can use $q(y)$ as a blanket over the target density $p(y)$. Let's demonstrate it with an example.

Example: Sampling from normal distribution

Consider $q(y)$ as a uniform distribution $U(0, 10)$ and the target density as a univariate normal distribution $\mathcal{N(5, 1)}$.

In the plot above, there is an M value ($M \approx 4$)such that $Mq(y) \geq p(y)$ for all $y$. Therefore, we can employ rejection sampling for this case. Now let's imagine a case where there is no such M.

Assume that the first sample from the $U(0, 10)$ turns out to be $3.825$.

$q(3.825) = 0.1$ just as any other value in $[0, 10]$ and we can obtain $p(3.825)$ since we know its pdf.

from scipy.stats import norm
normal_rv = norm(5, 1)
normal_rv.pdf(3.825)

0.20003737933848778

Obtaining $\frac {f(0.3825)} {Mg(0.3825)} \approx 0.5$,

import numpy as np
import matplotlib.pyplot as plt
from scipy.stats import norm
x = np.linspace(0, 10, 1000)
plt.figure(figsize=(10, 6))
plt.axhline(0.4, color="orange", label="$0.4q(y)$")

normal_rv = norm(5, 1)
plt.plot(x, normal_rv.pdf(x), linewidth=2, label="$p(y)$")
plt.xlim(0, 10)
plt.ylim(0, 0.48)
plt.legend();
plt.title("Density functions of $p(y)$, $0.4q(y)$ (unnormalized)")
plt.axvline(3.825, linestyle="--", linewidth=2, ymax=0.415)
plt.axvline(3.825, color="orange", linestyle="--", linewidth=2, ymin=0.415, ymax=0.830)
plt.axhline(0.2, color="black", linestyle="--", linewidth=2);

Dashed red lines show our first hypothetical sample from $U(0, 10)$. See the horizontal dashed line and observe that $0.4q(0.3825) = 0.4$ and $p(0.3825)=0.2$. Therefore, we should expect the probability of accepting the value $0.3825$ to be $0.5$:

$$Pr(accept| y = 3.825) \approx 0.5$$

In the next step we need to sample $u \sim U(0, 1)$ and accept the sampled $y$ value if

$u \leq \frac {f(x)} {Mg(x)} \approx 0.5$.

We know that $Pr(u \leq 0.5) = 0.5$ which means that $Pr(accept| y = 3.825) = 0.5$. This confirms our observation the on the above plot.

What if we can't find such $M$? The next plot demonstrates a $q(y)$ that can't be used to sample from $\mathcal{N(5, 1)}$.

In this case, we can't find a value $M$ such that $Mq(y) \geq p(y)$ for all $y$. No matter how large we choose the $M$, $Mq(y)$ will always be $0$ for $4 \leq x \leq 6$. This means that we will never be able to sample $x$ values between $4$ and $6$. Then the obtained samples will never follow our target distribution $\mathcal{N(5, 1)}$ independent of how we accept or reject the samples.

Obtain samples and compare with numpy random normal 

np.random.seed(0)
# Obtain 1000 samples from U[0, 10] (q(y))
q_samples = np.random.uniform(0, 10, 100000)

# Obtain 1000 samples from U[0, 1] (accept/reject criterion)
u_samples = np.random.uniform(0, 10, 100000)

# Obtain M. We know that q(x) = 0.1 for all x
M = normal_rv.pdf(5) / 0.1

# Accept the samples that satisfies the condition
norm_samples = q_samples[M * 0.1 * u_samples <= normal_rv.pdf(q_samples)]

You can see that most of the samples are rejected. This method is very inefficient in this case where the target distribution is normal and the samples are obtained from a uniform distribution.
